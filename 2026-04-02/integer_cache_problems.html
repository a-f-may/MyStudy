Integer キャッシュ範囲 確認問題（2回目：2026-04-02）

  • Java の Integer キャッシュ範囲は -128 〜 127
  • == はオブジェクトの参照比較、値比較には .equals() を使う
  • new Integer() は常に新しいオブジェクトを生成する

Q1

次のコードの出力を答えてください。

Integer a = 100;
Integer b = 100;
System.out.println(a == b);
System.out.println(a.equals(b));
解答を見る 
true
true

100 はキャッシュ範囲内なので a と b は同じオブジェクト。== も equals() も true。

Q2

次のコードの出力を答えてください。

Integer a = 128;
Integer b = 128;
System.out.println(a == b);
System.out.println(a.equals(b));
解答を見る 
false
true

128 はキャッシュ範囲外なので別オブジェクト。== は false、値は等しいので equals() は true。

Q3

次のコードの出力を答えてください。

Integer a = -128;
Integer b = -128;
System.out.println(a == b);

Integer x = -129;
Integer y = -129;
System.out.println(x == y);
解答を見る 
true
false

-128 はキャッシュ範囲の下限なので true。-129 は範囲外なので false。

Q4

次のコードの出力を答えてください。

Integer a = new Integer(100);
Integer b = new Integer(100);
System.out.println(a == b);
System.out.println(a.equals(b));
解答を見る 
false
true

new Integer() は常に新しいオブジェクトを生成するため、キャッシュ範囲内でも == は false。

Q5

次のコードの出力を答えてください。

Integer a = 127;
Integer b = 127;
Integer c = 128;
Integer d = 128;
System.out.println(a == b);
System.out.println(c == d);
System.out.println(a.equals(b));
System.out.println(c.equals(d));
解答を見る 
true
false
true
true

127 はキャッシュ上限なので == は true。128 は範囲外なので == は false。equals() は常に値比較なので両方 true。

Q6（総合）

空欄を埋めてください。

  • Java の Integer キャッシュ範囲は ___ 〜 ___
  • new Integer(100) == new Integer(100) の結果は ___
  • Integer の値比較には ___ メソッドを使うべき
解答を見る
  • -128 〜 127
  • false（常に新しいオブジェクト）
  • .equals()
