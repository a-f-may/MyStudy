## 問題 1：デバッグ問題

次のコードが「期待通りに動かない」と報告されました。原因を特定し修正してください。

```python
def check_user_age(age):
    """年齢が18以上かチェックする"""
    if age is 18:
        return "ちょうど18歳"
    elif age > 18:
        return "18歳以上"
    else:
        return "18歳未満"

# テスト
ages = [17, 18, 19, 30]
for a in ages:
    print(f"{a}歳: {check_user_age(a)}")

# 外部入力をシミュレート
user_age = int(input("年齢を入力: "))
result = check_user_age(user_age)
print(result)
```

<details>
<summary>解答を見る</summary>

**問題：** `age is 18` はキャッシュ範囲内（18 は -5〜256）なので、小さな数値ならたまたま動くことがありますが、`is` で値比較するのは誤りです。外部入力や大きな値では必ず失敗します。

**修正：**
```python
def check_user_age(age):
    if age == 18:      # is → == に変更
        return "ちょうど18歳"
    elif age > 18:
        return "18歳以上"
    else:
        return "18歳未満"
```

**補足：** `18` はキャッシュ範囲内なので今回は `is` でも動きますが、将来的に値が変わったりすると壊れる「時限爆弾」的なコードです。

</details>

## 問題 2：`id()` を使った確認

次のコードを実行したとき、`id` の値はそれぞれ同じになりますか？違いますか？

```python
import sys

for i in range(-10, 270):
    a = i
    b = i
    if a is not b:
        print(f"{i}: キャッシュ範囲外（id(a)={id(a)}, id(b)={id(b)})")
```

<details>
<summary>解答を見る</summary>

`-10` から `-6`（5個）と `257` から `269`（13個）の整数でキャッシュ範囲外のメッセージが表示されます。

ただし、**同じ行（同一コンテキスト）で代入した場合、Pythonの最適化により範囲外でも `is` が `True` になる場合があります**（CPythonの定数畳み込み最適化）。

実際に試して、環境による違いを確認してみましょう。

</details>

## 問題 3：実行環境による違い

次のコードはインタープリタで実行するか、スクリプトとして実行するかで結果が変わる場合があります。なぜですか？

```python
# このコードを .py ファイルとして実行した場合
a = 1000
b = 1000
print(a is b)   # Trueになることがある？
```

<details>
<summary>解答を見る</summary>

**理由：** CPythonはスクリプトファイルをコンパイルする際に**定数畳み込み（constant folding）最適化**を行います。同じスコープ内の `1000` というリテラルが同じ定数オブジェクトとして扱われるため、スクリプト実行時は `True` になることがあります。

一方、対話式インタープリタ（REPL）では各行が独立してコンパイルされるため、`False` になりやすいです。

**教訓：** `is` の動作は環境・実装に依存するため、整数の値比較には絶対に `==` を使うべきです。

</details>

## 問題 4：総合問題

次のコードの出力を予測してください。

```python
results = []
for i in [0, 100, 256, 257, 500, -5, -6]:
    a = i
    b = i
    results.append((i, a is b, a == b))

for num, is_same, eq in results:
    print(f"{num:4d}: is={is_same}, =={eq}")
```

<details>
<summary>解答を見る</summary>

```
   0: is=True,  ==True
 100: is=True,  ==True
 256: is=True,  ==True
 257: is=False, ==True   ※環境によりTrueの場合あり
 500: is=False, ==True   ※環境によりTrueの場合あり
  -5: is=True,  ==True
  -6: is=False, ==True   ※環境によりTrueの場合あり
```

`==` は常に `True`（値は等しい）。`is` はキャッシュ範囲内のみ確実に `True`。

</details>

## 問題 5：まとめクイズ

以下の主張について、正しい(○)か間違い(×)かを答えてください。

1. `a = 256; b = 256; a is b` は必ず `True` になる
2. `a = 257; b = 257; a is b` は必ず `False` になる
3. 整数の値を比較するには `==` より `is` を使う方が安全だ
4. `-5 is -5` は `True` になる
5. `is` 演算子はオブジェクトのIDを比較する

<details>
<summary>解答を見る</summary>

1. ○ — `256` はキャッシュ範囲の上限、必ず `True`
2. × — CPythonの最適化により、スクリプト内では `True` になる場合がある（実装依存）
3. × — `is` は値比較ではなくオブジェクト同一性チェック。値比較には `==` を使う
4. ○ — `-5` はキャッシュ範囲の下限、必ず `True`
5. ○ — `is` は `id(a) == id(b)` と等価

</details>
