while コンパイルエラーパターン（3回目：2026-04-04）

  • while(false) のブロック内は到達不可 → コンパイルエラー
  • break / return / throw の後のコードは到達不可 → コンパイルエラー
  • ループ外の break / continue → コンパイルエラー
  • while(true) の後のコードは到達不可 → コンパイルエラー

Q1

このコードはコンパイルエラーになりますか？

public void test() {
    while (false) {
        System.out.println("hello");
    }
}
解答を見る
コンパイルエラー：unreachable statement

while(false) の条件は常に false なので、ブロック内のコードは絶対に実行されません。Java はこれをコンパイル時に検出してエラーにします。

Q2

このコードはコンパイルエラーになりますか？

public void test() {
    while (true) {
        break;
        System.out.println("ここは？");
    }
}
解答を見る
コンパイルエラー：unreachable statement

break の後のコードは到達不可です。Java はこれをコンパイルエラーにします。

Q3

このコードはコンパイルエラーになりますか？

public void test() {
    int x = 5;
    break;
}
解答を見る
コンパイルエラー：break outside switch or loop

break はループまたは switch の中でしか使えません。

Q4

このコードはコンパイルエラーになりますか？

public void test() {
    while (true) {
        return;
    }
    System.out.println("終了");
}
解答を見る
コンパイルエラー：unreachable statement

while(true) からは抜け出せない（return でメソッド終了）ため、その後のコードは到達不可です。

Q5

このコードはコンパイルエラーになりますか？

public int getValue() {
    while (true) {
        // 何もしない
    }
}
解答を見る
コンパイルエラーにならない

Java は while(true) が絶対に終了しないことを認識しているため、return 文がなくてもコンパイルエラーにはなりません。

Q6

このコードはコンパイルエラーになりますか？

public void test() {
    int x = 5;
    continue;
}
解答を見る
コンパイルエラー：continue outside of loop

continue はループの中でしか使えません。

Q7

このコードはコンパイルエラーになりますか？

public void test() {
    while (true) {
        throw new RuntimeException();
    }
    System.out.println("終了");
}
解答を見る
コンパイルエラー：unreachable statement

while(true) 内で必ず throw されるため、ループの後のコードは到達不可です。

Q8（総合）

次の (a)〜(d) について、コンパイルエラーになるか答えてください。

// (a)
while (false) { int x = 0; }

// (b)
while (true) { break; int x = 0; }

// (c)
while (true) { }
int x = 0;

// (d)
while (true) { return; }
int x = 0;
解答を見る
  • (a) コンパイルエラー — while(false) のブロック内は到達不可
  • (b) コンパイルエラー — break の後の int x = 0 は到達不可
  • (c) コンパイルエラー — while(true) の後のコードは到達不可
  • (d) コンパイルエラー — while(true) { return; } の後は到達不可
