整数キャッシュ範囲 理解度確認問題（3回目：2026-04-04）

  • Pythonは -5 〜 256 の整数をキャッシュしている
  • == は値の比較、is は同一オブジェクトかの比較
  • 整数の値比較には必ず == を使うこと

Q1

次のコードの出力を答えてください。

a = 100
b = 100
print(a is b)
print(a == b)
解答を見る 
True
True

100 はキャッシュ範囲内なので a と b は同じオブジェクト。どちらも True。

Q2

次のコードの出力を答えてください。

a = 257
b = 257
print(a is b)
print(a == b)
解答を見る 
False
True

257 はキャッシュ範囲外。値は等しいが別オブジェクトなので is は False、== は True。

Q3

次のコードの出力を答えてください。

print(-5 is -5)
print(-6 is -6)
print(256 is 256)
print(257 is 257)
解答を見る 
True
False
True
False

-5 と 256 はキャッシュ範囲の境界値。-6 と 257 は範囲外。

Q4

次のコードの出力を答えてください。

def double(n):
    return n * 2

a = double(100)
b = double(100)
print(a is b)   # 200

x = double(150)
y = double(150)
print(x is y)   # 300
解答を見る 
True
False

200 はキャッシュ範囲内 → True。300 はキャッシュ範囲外 → False。関数の戻り値でもキャッシュ機構は変わらない。

Q5

次のコードにはバグがあります。何が問題で、どう直しますか？

def is_adult(age):
    if age is 18:
        return "ちょうど18歳"
    return "18歳ではない"

print(is_adult(18))
解答を見る

問題： is で整数の値を比較している。

修正：

def is_adult(age):
    if age == 18:   # is → ==
        return "ちょうど18歳"
    return "18歳ではない"

教訓： 整数の値比較には必ず == を使う。is はオブジェクト同一性チェック専用。
